akoeplinger commented on Jun 13, 2015

@hahoyer There are a two issues here:

Unable to find framework corresponding to the target framework moniker '.NETFramework,Version=v4.6'. Framework assembly references will be resolved from the GAC, which might not be the intended behavior.

Travis uses Mono to build C# projects and Mono doesn't ship the .NET 4.6 profile since it is not yet released and still in preview

warning : Could not find project
file /usr/lib/mono/xbuild/Microsoft/VisualStudio/v/VSSDK/Microsoft.VsSDK.targets, to import.

Mono doesn't ship the Visual Studio SDK. There are NuGet packages that might have what you need, but I've never used them so not sure if they're sufficient.

tim-cools commented on Nov 14, 2015

Locally I solved this problem by setting the mono TargetFrameworkVersion to v4.5 manually. Mono 4.5 supports the C# 6 syntax already... (Except for one line of code I had to change)

xbuild /p:Configuration=Release src/Soloco.ReactiveStarterKit.sln /p:TargetFrameworkVersion="v4.5"

On Trevis I get a CSC: error CS0041: Unexpected error writing debug information -- 'The requested feature is not implemented.'

akoeplinger commented on Nov 18, 2015

@tim-cools the error you're getting is because you're using Microsoft's new Roslyn C# compiler (which is brought into your project via some NuGet package) and it doesn't support writing debug information (.pdb files) on non-Windows platforms yet.

You can turn off debug symbols by passing /p:DebugSymbols=False in your build script.

Merwer commented on Apr 29, 2016

For anyone that comes across this and is using visual studio (I've been trying to solve this for hours), you can turn off the generation of PDB files by opening the properties for the csproj file; selecting the "build" tab; opening the "Advanced" modal; and selecting "Debug Info: None".
